Transforming Ersilia's CLI

One story of transforming a CLI-based machine-learning model hub

—Overview— What is Ersilia


Ersilia is a nonprofit organization based in Spain. Its mission is to develop and distribute machine learning models to aid infectious disease research. At the start of the project, Ersilia's model hub utilized a command-line interface (CLI). My team and I aimed to solve the following design problem: How could we redesign the CLI into an accessible GUI for scientists with resource constraints?

Role

—Our initial research— Framing the problem


User interviews


Our team first conducted 8 user interviews, where we spoke to our client as well as the users and developers of Ersilia's model hub. Through these interviews, we determined the following constraints:

  1. Many of the research scientists who used the model hub were less technically savvy
  2. Scientists had limited Internet access/connectivity since they were located in developing countries
  3. The client requested that the GUI be browser-based



Competitor analysis


Then, our team researched how other AI/ML model hubs were designed. During this step, we kept the following design principles in mind that our client wanted to emphasize:

  1. Simplicity: Reduce barrier to adoption with a clean front-end design
  2. Homogeneity: Standardize how models appear in the interface
  3. Discoverable models: Help users find relevant models easily
  4. Author credit: Give visibility to all original model authors
  5. Documentation: Build trust with users using sufficient documentation
  6. Scalability: Design should scale as number of models grow
  7. Sense of purpose: Users should feel connected with Ersilia's mission
  8. Proximity to the team: Users should feel welcome to provide feedback



—Gathering inspiration for designs— Our first iterations


Brainstorming & sketches


We first started with a brainstorming session, where we sketched out potential design ideas together. In this step, we agreed that we needed to accomplish the following in our design while still following the design principles above:

  1. Create an easy way for users to sort and filter for relevant models
  2. Present information like model info/results in an easily digestible manner
  3. Provide users resources to help them adjust to Ersilia's learning curve



Wireframing & low-fi prototype


From this step on, our group split the design into equal parts because of the relatively short project timeframe, and we each took responsbility for one part. However, we made sure to keep our designs with each other consistent and shared feedback with each other throughout the design process. I was in charge of designing the landing page of the model hub.

We built upon the ideas we had brainstormed in the previous step--for the landing page specifically, we considered aspects like filter location, colors, shapes, etc.



Mid-fi prototype


After a round of critique from our client, users, and colleagues, we created a higher fidelity prototype in Figma. In this version, our group decided that the landing page should:

  1. Have the filters on the left side of the page since we thought this would generate the most user familiarity.
  2. And the filters section as a whole and filters within this section should be expandible/collapsible due to the high number of tags that Ersilia utilizes.



—Our final design— Reflecting on the process & the hi-fi prototype


Conclusion & reflections


Designing for an unfamiliar field comes with many challenges: While some of my teammates had backgrounds in biology, all of us had to rapidly become comfortable with Ersilia's technical jargon and with exploring unknown territory.

Communication is essential in a team: This was my first time working closely with other designers. Open communication and trust was key in making sure that our design processes aligned in order to create a cohesive prototype.

What I would do with more time: If we had more time, I would have liked to explore the communicative aspect of the model hub and create other ways for users to interact with Ersilia's team since our client valued their bond with users.



Design solutions


Landing page:
  1. Expandable, color-coded filters + sorting system to find relevant models
  2. By default, most recent models are shown, so users can see what's new
  3. If no relevant models, users can see closest matches or contact Ersilia

Data modal:
  1. Defaults to using model online, emphasizing Ersilia's shift from CLI to GUI
  2. 2 tab format allows users to switch to the CLI in case of Internet issues
  3. Users are given relevant information without being overwhelmed

Model simulation:
  1. Ability to input multiple compounds to make model utilization easier
  2. Progress bar added so users know how long it will take for models to run
  3. Ability to copy outputs in order to feed this info as input into another model

Model hub guide:
  1. Approachable, easy to understand 3 step guide
  2. Available from landing page so users know about this resource
  3. Expand/collapse all button added for easy viewing
