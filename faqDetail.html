Jumbo Privacy User FAQ

A story of my ongoing work in front-end development of Jumbo's user FAQ.

Info

View the official FAQ page here.

The starting point:

Jumbo Privacy is a mobile app company, and the FAQ is available both through desktop and from within the app itself.

I took on the task of developing the FAQ when it was still in its very early stages. The following changes were done using HTML, CSS, and JavaScript/jQuery.

Beyond writing new articles and making small aesthetic changes, there were two areas I wanted to improve after conducting usability interviews:

  1. Optimizing navigation
    Individual articles were separated into discrete links. Users had to tap each link to view each article, then go back to the main list of questions in order to view another, making the process of viewing multiple questions tedious.

  2. Making the layout user friendly on mobile
    Tutorial images in the FAQ were small and hard to see without zooming in, especially for our older users.
Identifying and creating solutions:

  1. For my first goal, I knew I wanted users to be able to scroll through one page for their answers, minimizing any back-and-forth. But, I also didn't want to make the page overly long.

    My solution was to group articles into one page and make each one into a collapsible dropdown that could be expanded and condensed. This allowed users to easily pick and choose one or more questions to view without having to navigate to multiple pages.

  2. For the second, one possible workaround was displaying each image in its original size, making them easier to see. However, the main drawback was that the images would make the page too long and would require more scrolling. In addition, it wasn't aesthetically pleasing to have the tutorial images lined up vertically rather than horizontally in one row.

    I decided to create a lightbox image carousel that enlarged the images when tapped and also allowed users to follow the tutorial steps at the same time, eliminating the need to zoom in on each image individually.

Another QoL change:

After implementing the collapsible dropdowns, I realized that linking users to a specific question had become more difficult. Just providing the basic URL for each page meant that users then needed to search for the question they wanted answered.

To solve this, common article questions were assigned their own unique ID (#_____). Our team could then easily direct users to where they wanted to go by adjusting the ID included in the provided URL. Upon following the link, the user would automatically be brought to the exact article they needed on the page.

Final thoughts:

While the FAQ continues to evolve with other code changes, I've learned some things from my experience so far of working on the FAQ:

  • Implemented solutions can introduce new issues, but these obstacles are new opportunities to further better the product.
  • Always keep the end user and the context that they'll be using the product in when deciding what changes to make.
